# Designer Design Analysis Skill

## Overview
This skill provides guidance for analyzing design requirements and creating comprehensive design briefs as the first step in the UI design pipeline.

## When to Use
- When starting a new UI/UX design workflow
- When translating raw user descriptions into structured functional specifications
- When preparing sitemap and user flow diagrams

## Process

### 1. Requirement Extraction
- Identify user personas from the description
- Extract core features and functionality
- Note technical, timeline, and budget constraints
- Capture success metrics and KPIs

### 2. Structuring Information
- Organize findings into a cohesive design brief
- Identify page/feature requirements
- Document navigation patterns
- Note any ambiguities or assumptions

### 3. Validation
- Confirm all requirements are captured
- Check for conflicting constraints
- Ensure success metrics are measurable

## Output
A structured design brief document (see [design-brief.md](templates/design-brief.md)) saved to `.agents/plans/designer-output/briefs/[project-name]-brief.md`.

## Best Practices
- Ask clarifying questions when requirements are vague
- Document assumptions explicitly
- Prioritize features as "must-have" vs "nice-to-have"
- Consider mobile-first implications in early analysis
